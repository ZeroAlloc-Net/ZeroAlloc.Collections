# Cookbook: Custom Collection with Source Generator

This recipe shows how to build a domain-specific `OrderBuffer` collection using the `[ZeroAllocList]` and `[ZeroAllocEnumerable]` source generators, so you get a strongly-typed, pooled collection without writing boilerplate.

## The Problem

You have an `Order` type and need a specialized collection for it throughout your codebase. You could use `PooledList<Order>`, but:

1. `PooledList<T>` is a `ref struct` -- it cannot implement `IDisposable` for use in classes, DI-managed services, or async code.
2. You want a named type (`OrderBuffer`) that communicates intent and can carry domain-specific helper methods.
3. Writing the entire Add/Dispose/Enumerator boilerplate by hand for every domain type is tedious and error-prone.

## The Solution

ZeroAlloc.Collections ships two source generators:

| Attribute               | What It Generates                                                       |
|-------------------------|-------------------------------------------------------------------------|
| `[ZeroAllocList(typeof(T))]`    | A full `Add`/`Clear`/`Dispose`/`ToArray`/`AsSpan` list backed by `ArrayPool` |
| `[ZeroAllocEnumerable]`         | A `ref struct` enumerator with `ref readonly Current` for any type that has an array field and a count field |

`[ZeroAllocList]` generates a class (or struct) that implements `IDisposable`, so it works in fields, DI containers, and async methods -- unlike the `ref struct` `PooledList<T>`.

## Step 1: Define Your Domain Type

```csharp
namespace Trading;

public readonly record struct Order(
    long OrderId,
    string Symbol,
    decimal Price,
    int Quantity,
    DateTimeOffset Timestamp
);
```

## Step 2: Apply the Source Generator Attribute

```csharp
using ZeroAlloc.Collections;

namespace Trading;

/// <summary>
/// A pooled, strongly-typed buffer of Order values.
/// All list infrastructure is generated by the source generator.
/// </summary>
[ZeroAllocList(typeof(Order))]
public partial class OrderBuffer
{
    // The source generator fills in:
    //   - private Order[]? _array;
    //   - private int _count;
    //   - private ArrayPool<Order>? _pool;
    //   - Constructors (default, capacity, capacity+pool)
    //   - Count, indexer, Add, Clear, ToArray, AsSpan, AsReadOnlySpan
    //   - Dispose (returns buffer to pool)
    //   - GetEnumerator (ref struct enumerator with ref readonly Current)

    // You can add domain-specific methods on the partial class:
    public decimal TotalValue()
    {
        decimal sum = 0;
        foreach (ref readonly Order order in this)
        {
            sum += order.Price * order.Quantity;
        }
        return sum;
    }

    public bool ContainsSymbol(string symbol)
    {
        foreach (ref readonly Order order in this)
        {
            if (order.Symbol == symbol) return true;
        }
        return false;
    }
}
```

## What Gets Generated

The source generator emits a file named `OrderBuffer.ZeroAllocList.g.cs` with the following members:

```csharp
// <auto-generated />
public partial class OrderBuffer : System.IDisposable
{
    private Order[]? _array;
    private int _count;
    private ArrayPool<Order>? _pool;

    public OrderBuffer() { ... }                         // Uses ArrayPool<Order>.Shared
    public OrderBuffer(int capacity) { ... }             // Pre-sizes the buffer
    public OrderBuffer(int capacity, ArrayPool<Order> pool) { ... }

    public int Count { get; }
    public Order this[int index] { get; }                // Bounds-checked

    public void Add(Order item) { ... }                  // Grows automatically
    public void Clear() { ... }                          // Resets count, clears refs
    public Order[] ToArray() { ... }                     // Copies to new array
    public Span<Order> AsSpan() { ... }
    public ReadOnlySpan<Order> AsReadOnlySpan() { ... }

    public void Dispose() { ... }                        // Returns buffer to pool

    public Enumerator GetEnumerator() { ... }

    public ref struct Enumerator                         // Zero-alloc foreach
    {
        public ref readonly Order Current { get; }
        public bool MoveNext() { ... }
    }
}
```

Because `OrderBuffer` is a `class` (not `ref struct`), it implements `IDisposable` and can be used in fields, DI containers, and async methods.

## Step 3: Use ZeroAllocEnumerable for Existing Types

If you already have a type with an array field and a count field, `[ZeroAllocEnumerable]` generates just the enumerator:

```csharp
using ZeroAlloc.Collections;

namespace Trading;

/// <summary>
/// A manually managed collection that gets a zero-alloc enumerator
/// from the source generator.
/// </summary>
[ZeroAllocEnumerable]
public partial class TradeLog
{
    // The generator looks for the first array field and the first int field:
    private Trade[] _trades = new Trade[64];
    private int _count = 0;

    public void Record(Trade trade)
    {
        if (_count == _trades.Length)
        {
            Array.Resize(ref _trades, _trades.Length * 2);
        }
        _trades[_count++] = trade;
    }

    // The source generator adds:
    //   public Enumerator GetEnumerator() => new Enumerator(_trades, _count);
    //   public ref struct Enumerator { ... }
}

public readonly record struct Trade(string Symbol, decimal Price, int Qty);
```

## Complete Usage Example

```csharp
using Trading;

// OrderBuffer can be stored in fields and used across async boundaries
// because it is a class, not a ref struct.
public sealed class OrderProcessingService : IDisposable
{
    private readonly OrderBuffer _pendingOrders = new(capacity: 128);
    private readonly TradeLog _tradeLog = new();

    public void SubmitOrder(Order order)
    {
        _pendingOrders.Add(order);
    }

    public async Task FlushAsync(CancellationToken ct)
    {
        Console.WriteLine($"Flushing {_pendingOrders.Count} orders, total value: {_pendingOrders.TotalValue():C}");

        // Process each order — foreach uses the generated ref struct enumerator
        foreach (ref readonly Order order in _pendingOrders)
        {
            await ExecuteOrderAsync(order, ct);
            _tradeLog.Record(new Trade(order.Symbol, order.Price, order.Quantity));
        }

        _pendingOrders.Clear(); // Resets count, buffer is retained for reuse

        // Iterate the trade log — also uses a generated enumerator
        Console.WriteLine("Recent trades:");
        foreach (ref readonly Trade trade in _tradeLog)
        {
            Console.WriteLine($"  {trade.Symbol}: {trade.Qty} @ {trade.Price:C}");
        }
    }

    private static Task ExecuteOrderAsync(Order order, CancellationToken ct)
    {
        // Send to exchange, etc.
        return Task.CompletedTask;
    }

    public void Dispose()
    {
        // Returns the pooled array back to ArrayPool<Order>.Shared
        _pendingOrders.Dispose();
    }
}
```

## When to Use Which

| Scenario                                 | Use                          |
|------------------------------------------|------------------------------|
| Short-lived, stack-scoped collection     | `PooledList<T>` (ref struct) |
| Field, DI, or async-compatible collection| `[ZeroAllocList(typeof(T))]` (generates class) |
| Add enumerator to existing type          | `[ZeroAllocEnumerable]`      |
| Fixed-size, fully stack-allocated        | `FixedSizeList<T>` + `stackalloc` |

## Key Takeaways

- **`[ZeroAllocList(typeof(T))]`** generates a complete pooled list implementation on a `partial class` or `partial struct`. It handles `Add`, `Clear`, `Dispose`, indexer, `AsSpan`, `ToArray`, and a `ref struct` enumerator.
- **`[ZeroAllocEnumerable]`** generates just the `GetEnumerator()` method and `ref struct Enumerator` for any type that has an array field and an int count field. Use it when you manage the storage yourself.
- Generated classes implement `IDisposable`, so they work in fields, DI containers, and across `await` boundaries -- unlike `ref struct` collections.
- Add domain-specific methods (like `TotalValue()`, `ContainsSymbol()`) on the `partial class`. The generator fills in the infrastructure; you own the domain logic.
- The generated enumerator yields `ref readonly T`, so iterating large value types does not copy them.
